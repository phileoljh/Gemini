# Mode 2：Thesis Sandbox & Critique（論點沙盤推演與壓力測試）

## 模式定位

對使用者**既有的投資觀點或市場論述**進行結構化壓力測試。此模式不是從零開始建構分析，而是以「魔鬼代言人」角色，系統性地挑戰論點中的隱藏假設、傳導斷裂、時間軸盲點與流動性風險。

> [!IMPORTANT]
> 此模式的核心價值是**找出論點的弱點**，而非驗證其正確性。即使使用者的論點大致正確，也必須找出至少 2-3 個可能導致論點失效的情境。

---

## 工作流程

### 步驟 1：論點解構 (Thesis Decomposition)
1. 將使用者的投資論點拆解為可獨立驗證的**核心命題** (Core Propositions)。
2. 每個命題標記其**隱含假設** (Implicit Assumptions)。
3. 識別論點中**未提及的傳導環節**（即「跳過的步驟」）。

**範例**：
- 使用者論點：「聯準會降息後，TLT 會大漲」
- 核心命題 1：聯準會即將降息（假設：通膨已受控）
- 核心命題 2：降息 → 長端殖利率下降（假設：期限溢酬不會上升）
- 核心命題 3：長端殖利率下降 → TLT 價格上漲（假設：沒有信用事件導致流動性緊縮）
- 未提及環節：國債供給增加是否會抵銷降息對長端殖利率的壓降效果？

### 步驟 2：傳導路徑驗證
- 引用 `core-toolkit.md` → 工具 2（Mermaid 傳導圖）
- **Critique 聚焦**：不是畫「完整傳導圖」，而是畫出使用者**假設的傳導路徑**，然後逐一標記每個節點的「通暢度」
- 標記規則：
  - 🟢 **暢通**：歷史證據支持此傳導在當前環境下有效
  - 🟡 **有條件**：傳導成立需要額外條件（須明確列出）
  - 🔴 **阻塞/斷裂**：有結構性因素可能使此傳導失效

### 步驟 3：PESTEL 盲點掃描
- 引用 `core-toolkit.md` → 工具 1（PESTEL）
- **Critique 聚焦**：不做全面掃描，只聚焦於使用者論點中**遺漏的 PESTEL 面向**
- 問自己：使用者的論點是否只考慮了 Economic 面，卻忽略了 Political 或 Legal 面的風險？

### 步驟 4：Trilogy 壓力測試
- 引用 `core-toolkit.md` → 工具 3（三部曲時間軸）
- **Critique 聚焦**：針對使用者的時間軸假設，提出**替代情境** (Alternative Scenarios)
- 具體操作：
  1. 呈現使用者的「基準情境」三部曲
  2. 提出至少一個「灰犀牛情境」（機率 20-40% 的不利發展）
  3. 提出至少一個「黑天鵝情境」（機率 <10% 但影響巨大）

### 步驟 5：投資時鐘一致性檢驗
- 引用 `core-toolkit.md` → 工具 4（美林投資時鐘）
- **Critique 聚焦**：檢驗使用者的資產配置是否與當前週期定位一致
- 如果使用者在停滯性通膨象限配置了大量成長股，必須明確標記此矛盾

### 步驟 6：主動風險偵測
- 引用 `proactive-triggers.md`
- 特別關注哪些觸發器可能直接瓦解使用者的論點

### 報告結構

```
# [論點摘要] — 投資論點沙盤推演報告

## BOTTOM LINE
[論點的整體強度評估：強固 / 有條件成立 / 脆弱]

## 一、論點解構
### 核心命題
[命題清單 + 隱含假設]

### 未提及的傳導環節
[跳過的步驟清單]

## 二、傳導路徑驗證
[Mermaid 圖，標記 🟢🟡🔴 通暢度]

## 三、盲點掃描
[使用者遺漏的 PESTEL 面向]

## 四、替代情境矩陣
### 基準情境（使用者觀點）
[Trilogy 表格]

### 灰犀牛情境（機率 20-40%）
[Trilogy 表格 + 觸發條件]

### 黑天鵝情境（機率 <10%）
[Trilogy 表格 + 觸發條件]

## 五、配置一致性檢驗
[投資時鐘定位 vs 使用者配置的矛盾點]

## ⚠️ 主動風險偵測
[觸發器輸出，若有]

## 建議調整方案
[WHAT → 修正什麼 / WHY → 為什麼 / HOW TO ACT → 具體操作]
```

### 品質檢查清單
- [ ] 至少拆解出 3 個核心命題
- [ ] 每個命題都有標記隱含假設
- [ ] 傳導圖有至少一個 🔴 或 🟡 節點（完全 🟢 意味著沒有壓力測試價值）
- [ ] 提出了至少一個灰犀牛與一個黑天鵝情境
- [ ] 替代情境有具體的觸發條件，不是泛泛的「如果出問題」
